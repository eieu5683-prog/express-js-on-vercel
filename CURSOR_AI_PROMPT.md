# Cursor AI 프롬프트 모음

엑셀 데이터를 코드로 변환할 때 사용하는 프롬프트 모음입니다.

## KSIC 데이터 변환 프롬프트

### 기본 프롬프트

```
내가 제공하는 엑셀 데이터를 바탕으로 src/data/ksicData.ts 파일의 KSIC_DB 객체를 업데이트해줘.
데이터 형식은 우리가 정의한 KsicInfo 인터페이스를 엄격히 따라야 해.
엑셀의 각 행을 객체의 키(업종코드)와 값으로 변환해줘.
```

### 상세 프롬프트 (더 구체적인 지시)

```
내가 제공하는 엑셀 데이터를 바탕으로 src/data/ksicData.ts 파일의 KSIC_DB 객체를 업데이트해줘.

요구사항:
1. 데이터 형식은 src/data/types.ts에 정의된 KsicInfo 인터페이스를 엄격히 따라야 해
2. 엑셀의 각 행을 객체의 키(업종코드 5자리)와 값으로 변환해줘
3. PSST 데이터가 있는 경우 psst 필드에 PSSTContent 형식으로 변환해줘
4. AI 표기법 준수 문구는 자동으로 추가하지 말고, 원본 데이터 그대로 사용해줘
5. 기존 데이터는 덮어쓰지 말고 추가만 해줘

엑셀 데이터 형식:
- 컬럼: 업종코드 | 업종명 | 등급 | 지원금액 | 솔루션 | P | S | S_scale | T
- 구분자: | (파이프)
```

### PSST 데이터 포함 프롬프트

```
엑셀 데이터를 KSIC_DB에 추가해줘. PSST 데이터가 포함된 경우 다음 형식으로 변환해줘:

psst: {
  p: "Problem 내용",
  s: "Solution 내용",
  s_scale: "Scale-up 내용",
  t: "Team 내용"
}

각 필드는 엑셀의 P, S, S_scale, T 컬럼에서 가져와야 해.
```

## 일정 데이터 변환 프롬프트

```
내가 제공하는 공고 일정 엑셀 데이터를 src/data/scheduleData.ts 파일의 SCHEDULE_DB 배열을 업데이트해줘.

요구사항:
1. 데이터 형식은 YearlySchedule 인터페이스를 따라야 해
2. 연도별로 그룹화하여 저장해줘
3. 날짜 형식은 YYYY-MM-DD 형식으로 변환해줘
4. targetIndustries는 배열로 변환해줘 (쉼표로 구분된 경우)

엑셀 데이터 형식:
- 컬럼: 연도 | 공고명 | 시작일 | 종료일 | 지원금액 | 대상업종 | 링크 | 비고
```

## 사용 예시

### 예시 1: 기본 KSIC 데이터 변환

1. 엑셀에서 데이터 복사:
```
58221 | 시스템 소프트웨어 개발 | S | 최대 1.5억 | R&D 가점 높음 | P:기존 수작업... | S:AI 자동화... | S_scale:3년 내... | T:전문가 구성...
```

2. Cursor AI 채팅창에 프롬프트 입력:
```
내가 제공하는 엑셀 데이터를 바탕으로 src/data/ksicData.ts 파일의 KSIC_DB 객체를 업데이트해줘.
데이터 형식은 우리가 정의한 KsicInfo 인터페이스를 엄격히 따라야 해.
엑셀의 각 행을 객체의 키(업종코드)와 값으로 변환해줘.
```

3. 엑셀 데이터 붙여넣기

4. 결과 확인:
```typescript
'58221': {
  code: '58221',
  name: '시스템 소프트웨어 개발',
  grade: 'S',
  maxAmount: '최대 1.5억',
  solution: 'R&D 가점 높음',
  psst: {
    p: '기존 수작업...',
    s: 'AI 자동화...',
    s_scale: '3년 내...',
    t: '전문가 구성...',
  },
}
```

### 예시 2: 일정 데이터 변환

1. 엑셀에서 데이터 복사:
```
2026 | 예비창업패키지 | 2026-01-15 | 2026-02-28 | 최대 1억원 | IT/소프트웨어,바이오,제조업 | https://example.com | 신규 창업자 대상
```

2. Cursor AI 채팅창에 프롬프트 입력:
```
내가 제공하는 공고 일정 엑셀 데이터를 src/data/scheduleData.ts 파일의 SCHEDULE_DB 배열을 업데이트해줘.
```

3. 엑셀 데이터 붙여넣기

## 문제 해결

### 변환이 제대로 안 될 때

1. **엑셀 데이터 형식 확인**
   - 구분자가 올바른지 확인 (| 또는 탭)
   - 컬럼 순서가 올바른지 확인

2. **프롬프트를 더 구체적으로 작성**
   - 데이터 형식 명시
   - 변환 규칙 명시

3. **수동 변환 고려**
   - 데이터가 적은 경우 수동으로 입력
   - 복잡한 경우 단계별로 변환

### 타입 오류 발생 시

1. `src/data/types.ts`의 타입 정의 확인
2. 변환된 데이터가 타입을 따르는지 확인
3. 필요한 경우 타입 정의 수정

## 팁

- **대량 데이터**: 여러 번에 나눠서 변환
- **복잡한 데이터**: 단계별로 변환 (먼저 기본 정보, 그 다음 PSST 데이터)
- **검증**: 변환 후 항상 타입 오류 확인
